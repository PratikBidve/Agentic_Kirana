# Local Development Guide

This file explains how to run **Agentic Kirana** locally, what each service does, and the exact day-to-day development workflow.

---

## Stack Overview

This repo has two runtime parts that must run simultaneously:

**1. Hono + Bun API Gateway** (port `3000`)
- Runs on Cloudflare Workers locally via `wrangler dev`
- Handles auth, stores, inventory, khata, and agent job ingestion
- Uses PostgreSQL via Drizzle ORM (Neon HTTP driver)
- Pushes AI jobs to Redis queue
- Validates all inputs with Zod

**2. FastAPI + LangGraph Worker** (port `8000`)
- Runs as a separate Python service
- Listens for queued jobs via Redis `BRPOP`
- Processes LangGraph AI agent DAG
- Writes results back to PostgreSQL

**Supporting infra:**
- **PostgreSQL 17** — application data (port `5432`)
- **Redis 7** — job queue + rate limiting (port `6379`)

## First-Time Setup

### Step 1 — Clone

```bash
git clone https://github.com/PratikBidve/Agentic_Kirana.git
cd Agentic_Kirana
```

### Step 2 — Install JS dependencies

```bash
bun install
```

### Step 3 — Create env file

```bash
cp .env.example .env
```

Open `.env` and fill in all values:

```env
NODE_ENV=development

# Local Docker Postgres
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/kirana

# Local Docker Redis
REDIS_URL=redis://localhost:6379

# Google OAuth — get from https://console.cloud.google.com
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

# Better Auth — any random string >= 32 chars
BETTER_AUTH_SECRET=a_very_long_random_secret_string_here

# This must match the port Wrangler runs on
BETTER_AUTH_URL=http://localhost:3000

# Your frontend URL for CORS
ALLOWED_ORIGIN=http://localhost:3001
```

> **Important:** Wrangler dev server uses `.dev.vars` for runtime bindings.
> Copy `.env` to `.dev.vars` as well:

```bash
cp .env .dev.vars
```

### Step 4 — Start Docker infrastructure

```bash
bun run docker:up
```

This starts:
- **kirana_db** — PostgreSQL 17 on port `5432`
- **kirana_redis** — Redis 7 (Alpine) on port `6379`

Verify both are healthy:

```bash
docker ps
```

You should see both containers with status `healthy`.

### Step 5 — Push database schema

```bash
bun run db:push
```

This creates all tables: `user`, `session`, `account`, `verification`, `store`, `product`, `customer`, `khata_entry`, `agent_job`.

Verify tables were created:

```bash
bun run db:studio
```

This opens Drizzle Studio in your browser where you can browse all tables.

### Step 6 — Start the Hono API gateway

```bash
bun run dev
```

Api is available at `http://localhost:3000`.

Test it:

```bash
curl http://localhost:3000/health
# Expected: {"status":"ok","timestamp":"..."}
```

Better Auth OpenAPI docs:
```
http://localhost:3000/api/auth/reference
```

## Python Worker Setup

Open a **new terminal** in the project root:

```bash
cd python_worker
```

**Create a Python virtual environment:**

```bash
# macOS / Linux
python -m venv .venv
source .venv/bin/activate

# Windows PowerShell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

You should see `(.venv)` in your prompt.

**Install dependencies:**

```bash
pip install -r requirements.txt
```

**Create worker env:**

```bash
cp .env.example .env
```

Edit `python_worker/.env`:

```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/kirana
REDIS_URL=redis://localhost:6379
OPENAI_API_KEY=sk-your-key-here
```

**Start the worker:**

```bash
uvicorn main:app --reload --port 8000
```

Verify:

```bash
curl http://localhost:8000/health
# Expected: {"status":"ok","worker":"running"}
```

The worker logs will show:
```
INFO: Kirana worker started. Listening on Redis queue: agent_jobs
```


## How to Build New Features (Correct Order)

Always follow this order to keep the codebase maintainable:

```
1. Schema     →  src/db/schema.ts + bun run db:push
2. Route      →  src/routes/<domain>/<name>.route.ts
3. Register   →  src/app.ts
4. Test       →  tests/<name>.test.ts
5. Docs       →  docs/API.md
6. Worker     →  python_worker/main.py (only if AI processing needed)
```

**Never write AI logic first.** Build the DB layer and HTTP routes first, confirm they work, then add agent orchestration.


## Common Errors and Fixes

### `Cannot find module 'bun:test'`

```bash
bun install
```

Check `tsconfig.json` has:

```json
{ "compilerOptions": { "types": ["bun-types"] } }
```

### `Cannot find type definition file for 'bun-types'`

`bun-types` was missing from `devDependencies`. Run:

```bash
bun install
```

### Postgres connection refused

```bash
bun run docker:up
docker ps   # verify kirana_db shows "healthy"
```

### Schema tables missing

```bash
bun run db:push
bun run db:studio   # verify tables exist
```

### Redis not processing agent jobs

Check:
1. `bun run docker:up` — kirana_redis must be healthy
2. Python worker must be running on port 8000
3. `python_worker/.env` must have correct `REDIS_URL`

Check queue:

```bash
docker exec -it kirana_redis redis-cli LLEN agent_jobs
```

### Auth not working locally

Check:
- Google OAuth callback URL is `http://localhost:3000/api/auth/callback/google` in Google Console
- `.dev.vars` file exists and has `BETTER_AUTH_URL=http://localhost:3000`
- `ALLOWED_ORIGIN` matches your frontend port

### Wrangler uses `.dev.vars` not `.env`

`wrangler dev` reads env bindings from `.dev.vars`, not `.env`.
If your app can't find env vars at runtime, run:

```bash
cp .env .dev.vars
```

## Testing

All tests are in `tests/` and use the native Bun test runner.

```bash
bun test
```

Current test coverage:

| File | What it tests |
|---|---|
| `tests/store.test.ts` | Auth guard, GST validation, mobile format |
| `tests/inventory.test.ts` | Auth guard, unit enum, negative price |
| `tests/khata.test.ts` | Auth guard, zero-amount, confirm endpoint |
| `tests/agent.test.ts` | Missing storeId, empty message, 2000 char limit |

To run a single test file:

```bash
bun test tests/agent.test.ts
```

To run in watch mode during development:

```bash
bun run test:watch
```

## Production Deployment Notes

- Replace `DATABASE_URL` with a [Neon](https://neon.tech) connection string — the Neon HTTP driver is already in use
- Replace `REDIS_URL` with [Upstash Redis](https://upstash.com) for serverless-compatible Redis
- Set `ALLOWED_ORIGIN` to your production frontend URL
- Set `NODE_ENV=production`
- Deploy Hono: `bun run deploy`
- Deploy Python worker on Railway, Render, or any container host
- Set Cloudflare Worker secrets via `wrangler secret put <KEY>`
